# Claude Code 設定

## 言語設定
- コードコメントは日本語
- ドキュメント（`docs/`）は**英語**で記述する（public リポジトリのため）

---

## ドキュメント管理ルール

### `docs/` の構成

```
docs/
  api-spec.md       # API 仕様（公開）
  TESTING.md        # テスト説明（公開）
  internal/         # 内部設計・議論（.gitignore 対象、非公開）
    architecture.md
    roadmap.md
    prov-dm-design.md
    ...
```

### 新しいドキュメントを作るとき

| 内容 | 置き場所 |
|------|---------|
| 外部ユーザー向け（API・使い方・テスト方法） | `docs/` 直下（英語） |
| 内部設計・意思決定・議論・ロードマップ | `docs/internal/`（日本語可） |

> `docs/internal/` は `.gitignore` に登録されており、public リポジトリには push されない。

---

## AI の行動ルール

**やること**
- 変更前にファイルを必ず読む
- セキュリティを常に考慮して実装する

**やらないこと**
- シークレット・API キーをコードにハードコードする
- 不要なファイル・ディレクトリを作成する
- 聞かれていないことを勝手に変更する
